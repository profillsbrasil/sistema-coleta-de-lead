## Authentication & Identity

**Auth Provider:**
- Better-Auth 1.5.5 (custom implementation, not OAuth)
- Database-backed sessions with email/password support

**Authentication Flow:**
- Email/password registration and login enabled (`emailAndPassword: { enabled: true }`)
- Session tokens stored in `session` table with expiry tracking
- Cookies via `nextCookies()` plugin for Next.js
- CORS-aware: Trusts origin from `CORS_ORIGIN` env var

**Session Management:**
- Location: `packages/auth/src/index.ts`
- Adapter: Drizzle with PostgreSQL provider
- Session tables: `user`, `session`, `account`, `verification`
- Session retrieval: Via `auth.api.getSession({ headers })` in API context

**Authentication Context:**
- File: `packages/api/src/context.ts`
- Pattern: Extract session from request headers in each API call
- Type: `Context = { auth: null, session?: Session }`

**Protected Routes:**
- Pattern: `protectedProcedure` in tRPC routers
- Throws `UNAUTHORIZED` error if `ctx.session` is missing
- Location: `packages/api/src/index.ts`

**Frontend Auth Integration:**
- Currently minimal; no explicit sign-out UI in reviewed code
- User session available in `ctx.session.user` in protected procedures
- Cookie credentials sent automatically via `credentials: "include"` in tRPC client

## Data Flow

**Client to Server:**
1. React component calls `trpc.todo.create.useMutation()`
2. TanStack React Query queues mutation
3. tRPC client sends POST to `/api/trpc` with batch payload
4. Next.js route handler: `packages/api/src/routers/todo.ts`
5. Procedure validates input via Zod schema
6. Database mutation via Drizzle: `db.insert(todo).values()`
7. PostgreSQL executes insert
8. Result returned to client, React Query refetch triggered
9. Component re-renders with new data

**Authentication Injection:**
1. Request arrives at `/api/auth/[...all]` or `/api/trpc/[trpc]`
2. Better-Auth middleware extracts session from cookies
3. Context created: `createContext(req: NextRequest)`
4. `auth.api.getSession({ headers: req.headers })` checks cookie
5. Session passed to tRPC router context
6. Protected procedures verify `ctx.session` exists
