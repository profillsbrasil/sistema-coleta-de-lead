## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` oklch(1 0 0) | Fundo da pagina de login |
| Secondary (30%) | `var(--card)` oklch(1 0 0) | Card de login, card-foreground para texto |
| Accent (10%) | `var(--primary)` oklch(0.511 0.096 186.391) | Botao Google (CTA principal), focus rings, link do user-menu |
| Destructive | `var(--destructive)` oklch(0.577 0.245 27.325) | Erro de autenticacao (toast) |

Accent reserved for:
- Botao de "Entrar com Google" (variant `default` = primary)
- Focus ring dos botoes (`:focus-visible`)
- Link ativo no user-menu

Botoes secundarios (LinkedIn, Facebook): usam variant `outline` — nao consomem accent color.

Dark mode: todas as variaveis ja tem equivalentes definidos em `globals.css` `.dark {}`. Nenhuma cor adicional necessaria.

## Interaction Contract

### OAuth Flow

1. Usuario clica em botao de provider
2. Botao clicado: spinner substitui icone, label permanece, demais botoes ficam `disabled`
3. Redirect para OAuth provider (Supabase Auth gerencia)
4. Retorno via `/auth/callback` (route handler, sem UI)
5. Sucesso: redirect para `/dashboard`
6. Erro: redirect para `/login?error=auth-code-error`, toast de erro renderizado

### Route Protection

1. Proxy.ts intercepta requests
2. Nao autenticado em rota protegida: redirect para `/login`
3. Vendedor em rota admin: tRPC retorna 403, UI mostra toast de erro (nao precisa de pagina 403 dedicada nesta fase)

### User Menu

1. Click no avatar/nome abre DropdownMenu
2. Opcao "Sair" executa `supabase.auth.signOut()` + redirect para `/login`
3. Sem confirmacao para logout (acao reversivel)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Botoes OAuth | `<button>` semantico via Base UI Button, label descritivo ("Entrar com Google") |
| Focus order | Tab order natural: Google -> LinkedIn -> Facebook |
| Focus ring | `focus-visible:ring-3 focus-visible:ring-ring/50` (ja presente no Button) |
| Icones | Decorativo (aria-hidden), label no texto do botao e suficiente |
| Loading state | `aria-busy="true"` no botao em loading, `disabled` nos demais |
| Error toast | Sonner usa `role="status"` por default (live region) |
| Color contrast | oklch primary sobre primary-foreground: ratio >= 4.5:1 (verificar pos-implementacao) |
| Touch target | Botoes size `lg` = h-9 (36px) — minimo aceitavel; width full compensa |
