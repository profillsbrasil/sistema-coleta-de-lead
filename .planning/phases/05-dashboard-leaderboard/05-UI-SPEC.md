## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / Label | 14px | 400 (regular) | 1.5 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |
| Caption | 12px | 400 (regular) | 1.4 |

Notes:
- Display (28px): numero principal dos stat cards (Total Leads, Score). Peso 600 para destaque numerico.
- Heading (20px): titulo da pagina, titulos de secao ("Leads por Tag").
- Body (14px): labels dos stat cards ("Total", "Quentes"), nome do vendedor no leaderboard.
- Caption (12px): staleness indicator ("Atualizado ha 5 min"), posicao no ranking ("#1"), subtitulos secundarios.
- Weights: 2 pesos -- 400 (regular) para body/label/caption, 600 (semibold) para headings/display/numeros destaque.

Source: Phase 1 tokens mantidos. Display (28px) usado pela primeira vez nesta fase para stat cards.

### StatCard

```
+---------------------------+
|  [label]      14px/400    |
|  [value]      28px/600    |
+---------------------------+
```

Constraints:
- Card padding: 16px (`p-4`)
- Layout: `flex flex-col gap-1`
- Label: Body (14px/400) `text-muted-foreground`
- Value: Display (28px/600) `text-foreground`
- Grid dos stat cards: `grid grid-cols-2 gap-2` para Total + Hoje
- Grid tags: `grid grid-cols-3 gap-2` para Quente + Morno + Frio
- Stat cards de tag: label colorido com cor oklch da tag, value em `text-foreground`
- Score card: full-width, mesma estrutura do StatCard

### LeaderboardEntry

```
+----------------------------------------------+
|  #1     Maria Silva                          |
|         52 leads            186 pts          |
+----------------------------------------------+
```

Constraints:
- Card padding: 16px (`p-4`)
- Layout: `flex items-start gap-3`
- Rank: Caption (12px/400) `text-muted-foreground`, min-width 24px
- Nome: Body (14px/600 semibold)
- Segunda linha: `flex justify-between w-full`
- Total leads: Caption (12px/400) `text-muted-foreground` -- "{n} leads"
- Score: Body (14px/600 semibold) -- "{n} pts"
- Card do vendedor logado: `border-2 border-primary bg-primary/5`
- Cards dos outros: `border border-border`
- Gap vertical entre entries: 8px (`gap-2`)
- Lista completa sem paginacao (equipe pequena, ~10 vendedores)

### Bar Chart (Recharts via shadcn Chart)

Constraints:
- Container: Card com padding 16px
- Titulo "Leads por Tag": Body (14px/600 semibold) acima do chart
- Chart type: `BarChart` horizontal (Recharts `layout="vertical"`)
- 3 barras: Quente, Morno, Frio
- Cores das barras: mesmas cores oklch de text das tags (light mode)
- Chart height: 120px (3 barras compactas)
- ChartTooltip com shadcn ChartTooltipContent
- ChartConfig com labels: `{ quente: { label: "Quente" }, morno: { label: "Morno" }, frio: { label: "Frio" } }`

### States

| State | Visual |
|-------|--------|
| Dashboard carregando | 5 Skeleton cards no grid (2 + 3) + 1 skeleton chart |
| Dashboard com dados | Stat cards preenchidos + chart |
| Dashboard sem leads | Stat cards com valor "0", chart vazio com mensagem inline |
| Leaderboard carregando (1o fetch) | 3 Skeleton entries empilhados |
| Leaderboard com dados | Lista de entries com highlight do vendedor logado |
| Leaderboard cache offline | Dados da ultima sync + staleness indicator visivel |
| Leaderboard sem cache (1o acesso offline) | Mensagem inline: "Conecte-se para ver o ranking" |
| Leaderboard erro fetch | Toast de erro + mostra cache se disponivel, ou mensagem vazia |
| Offline | Dashboard pessoal: comportamento identico (Dexie local). Leaderboard: mostra cache + staleness. |

## Interaction Contract

### Navegacao entre tabs

1. Vendedor acessa `/dashboard` (home apos login)
2. Tab "Meu Dashboard" e a tab ativa por default
3. Vendedor toca em "Leaderboard" para ver o ranking
4. Mudanca de tab e instantanea (client-side, sem rede)
5. Tab ativa persiste durante a sessao via estado local (nao URL)

### Dashboard pessoal

1. Ao montar, `useLiveQuery` busca todos os leads do vendedor no Dexie
2. Stat cards preenchidos instantaneamente (dados locais)
3. Chart de barras renderizado com breakdown por tag
4. Score calculado client-side: `(quentes * 3) + (mornos * 2) + (frios * 1)`
5. Dados reativos: criar/editar/excluir lead em outra tab atualiza stats automaticamente

### Leaderboard

1. Ao montar, tenta fetch via tRPC: `leaderboard.getRanking`
2. Se online: recebe ranking, grava no Dexie `leaderboardCache`, renderiza
3. Se offline: le do Dexie `leaderboardCache`, renderiza com staleness indicator
4. Staleness indicator mostra tempo desde ultimo fetch bem-sucedido
5. Vendedor logado identificado pelo userId -- card destacado com borda primary
6. Lista ordenada por score desc, empate desempatado por total leads desc
7. Se nenhum cache existe e esta offline: mostra mensagem de empty state

### Integracao com sync

1. Sync engine roda normalmente (push/pull leads)
2. Apos pullChanges bem-sucedido, fetch do leaderboard e disparado
3. Leaderboard atualizado no Dexie `leaderboardCache` com timestamp
4. Se fetch do leaderboard falha, sync de leads nao e afetado (independente)

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tabs | shadcn Tabs com `role="tablist"`, `role="tab"`, `role="tabpanel"` (built-in via Base UI) |
| Tab keyboard | Arrow keys navegam entre tabs, Enter/Space ativa (built-in) |
| Stat cards | Semantico: `<div>` com label visivel, valor como texto. Nao interativo. |
| Chart | `aria-label="Grafico de leads por tag"` no container. Dados tambem visiveis nos stat cards (redundancia acessivel). |
| Leaderboard list | `<ol>` semantico para lista ordenada. Cada entry como `<li>`. |
| Current user entry | `aria-current="true"` no entry do vendedor logado |
| Staleness indicator | `aria-live="polite"` para atualizar timestamp sem interromper |
| Loading skeleton | `aria-busy="true"` no container durante loading |
| Focus order (dashboard) | Tab: Tabs triggers -> stat cards (nao focaveis) -> chart -> FAB |
| Focus order (leaderboard) | Tab: Tabs triggers -> leaderboard entries (nao focaveis) -> FAB |
| Color contrast | Numeros Display em `text-foreground` sobre `card` background: ratio >= 4.5:1. Tags oklch calibradas (verificar pos-implementacao). |
