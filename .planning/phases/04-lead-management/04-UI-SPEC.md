## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body / Label | 14px | 400 (regular) | 1.5 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Body (14px): nome do lead no card, texto de contato (telefone/email).
- Caption (12px): timestamp relativo ("ha 5 min") no card. Usa `text-xs` do Tailwind.
- Heading (20px): titulo da pagina "Meus Leads", titulo "Editar Lead" na pagina de detalhe.
- Display (28px): nao usado nesta fase.
- Weights: 2 pesos -- 400 (regular) para body/label/caption, 600 (semibold) para headings.

Source: Phase 1 tokens mantidos. Caption (12px) adicionado para timestamp relativo nos cards.

## Layout Contract

### Pagina `/leads` (Lista de Leads)

```
+------------------------------------------+
|  [Header com nav]                        |
+------------------------------------------+
|                                          |
|  Meus Leads                     (count)  |
|                                          |
|  [Todos] [Quente] [Morno] [Frio]        |
|                                          |
|  +------------------------------------+  |
|  | Nome do Lead              [Quente] |  |
|  | (11) 99999-0000          ha 5 min  |  |
|  +------------------------------------+  |
|                                          |
|  +------------------------------------+  |
|  | Outro Lead                 [Morno] |  |
|  | email@test.com            ha 2h    |  |
|  +------------------------------------+  |
|                                          |
|  ... (infinite scroll)                   |
|                                          |
|  [sentinel div - invisible]              |
|                                          |
|                              +--------+  |
|                              |  [+]   |  |
|                              +--------+  |
+------------------------------------------+
```

Constraints:
- Lista max-width: 480px (`max-w-[480px] w-full mx-auto`)
- Padding horizontal mobile: 16px (`px-4`)
- Gap vertical entre cards: 16px (`gap-4`)
- Titulo "Meus Leads": Heading (20px/600), alinhado a esquerda
- Counter de leads: Body (14px/400), cor `muted-foreground`, alinhado a direita do titulo
- TagFilter: posicionado abaixo do titulo, gap 16px (`mt-4`)
- Lista: posicionada abaixo do filtro, gap 24px (`mt-6`)
- FAB: reutilizado da Phase 3 (`fixed bottom-6 right-6`, 56x56px, `z-50`)

### LeadCard (Item da lista)

```
+----------------------------------------------+
|  Nome do Lead                       [badge]  |
|  (11) 99999-0000                   ha 5 min  |
+----------------------------------------------+
```

Constraints:
- Card padding: 16px (`p-4`)
- Layout: `flex items-center justify-between`
- Lado esquerdo: nome (14px/600 semibold) + contato (14px/400 `text-muted-foreground`)
- Lado direito: tag badge (12px oklch) + timestamp relativo (12px `text-muted-foreground`)
- Lado direito: `flex flex-col items-end gap-1`
- Cursor: `cursor-pointer`
- Hover: `hover:bg-muted` (feedback visual de que e clicavel)
- Tag badge: `inline-flex items-center rounded-md px-2 py-0.5 text-xs font-medium` com cores oklch
- Entire card clicavel via `onClick` (navega para `/leads/[id]`)
- `role="button"` e `tabIndex={0}` para acessibilidade teclado
- `onKeyDown` handler: Enter e Space ativam navegacao

### Pagina `/leads/[id]` (Detalhe/Edicao)

```
+------------------------------------------+
|  <- Voltar          Editar Lead          |
+------------------------------------------+
|                                          |
|  [LeadForm em modo edicao]              |
|  (campos pre-preenchidos com dados      |
|   do lead existente)                     |
|                                          |
|  [====== Salvar alteracoes ======]      |
|                                          |
|  [Excluir Lead] (destructive, outline)  |
|                                          |
+------------------------------------------+
```

Constraints:
- Reutiliza LeadForm da Phase 3 com prop `lead` para pre-preencher
- Header: botao voltar (icone `ArrowLeft`, `aria-label="Voltar"`) + titulo "Editar Lead" (20px/600)
- Botao "Salvar alteracoes": full-width, variant `default`, size `lg`
- Botao "Excluir Lead": full-width, variant `destructive`, variant visual `outline`, size `lg`, abaixo do botao salvar com gap 16px
- O botao "Excluir Lead" abre AlertDialog (nao executa direto)
- Card max-width: 480px (`max-w-[480px] w-full mx-auto`)

### States

| State | Visual |
|-------|--------|
| Lista carregando (1o render) | 3 Skeleton cards empilhados (mesma altura de um LeadCard: ~72px) |
| Lista com dados | Cards empilhados com infinite scroll |
| Lista vazia (sem leads) | Empty state centralizado: icone + titulo + descricao + CTA |
| Lista vazia (filtro ativo) | Mensagem inline: "Nenhum lead {tag} encontrado" |
| Infinite scroll loading | Spinner pequeno (Loader) abaixo do ultimo card, acima do sentinel |
| Sem mais leads | Sentinel invisivel, sem indicador (lista simplesmente para de crescer) |
| Edit: carregando lead | Skeleton do formulario (campos placeholder) |
| Edit: lead nao encontrado | Mensagem de erro + botao voltar |
| Edit: salvando | Botao "Salvar alteracoes" com spinner, disabled. Campos readonly. |
| Edit: sucesso | Toast "Lead atualizado!" + `router.back()` |
| Delete: confirmacao | AlertDialog aberto com titulo, descricao e dois botoes |
| Delete: sucesso | Toast "Lead excluido!" + redirect para `/leads` |
| Offline | Comportamento identico ao online (tudo via Dexie). Sem indicador visual nesta fase. |

## Interaction Contract

### Lista de leads

1. Vendedor acessa `/leads` (via header nav ou FAB no dashboard)
2. Lista carrega 20 leads do Dexie via `useLiveQuery` (instantaneo, offline)
3. Leads exibidos como cards empilhados, ordenados por `createdAt` desc
4. Scroll atinge sentinel -> carrega mais 20 leads (expanding limit)
5. Toque em card -> navega para `/leads/[id]`

### Filtro por tag

1. Vendedor toca em "Quente", "Morno" ou "Frio" no TagFilter
2. Filtro aplica instantaneamente via Dexie query (sem rede)
3. Toggle unico: clicar no filtro ativo volta para "Todos"
4. "Todos" e o default ao entrar na pagina
5. Se filtro ativo resulta em lista vazia, mostra mensagem inline

### Edicao de lead

1. Vendedor toca no LeadCard -> navega para `/leads/[id]`
2. LeadForm carrega em modo edicao (campos pre-preenchidos)
3. Vendedor edita campos desejados
4. Toca "Salvar alteracoes" -> `updateLead()` grava no Dexie + syncQueue
5. Toast "Lead atualizado!" + `router.back()`

### Exclusao de lead

1. Na pagina `/leads/[id]`, vendedor toca "Excluir Lead"
2. AlertDialog abre com confirmacao
3. Vendedor toca "Excluir" no dialog -> `deleteLead()` seta `deletedAt` no Dexie + syncQueue
4. Toast "Lead excluido!" + redirect para `/leads`
5. Lead desaparece da lista (filtro `deletedAt === null` no query)

### Navegacao

- Header nav inclui link para `/leads`
- FAB na pagina `/leads` navega para `/leads/new` (Phase 3)
- Botao voltar na pagina de edicao usa `router.back()`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| LeadCard | `role="button"`, `tabIndex={0}`, `onKeyDown` para Enter/Space, `cursor-pointer` |
| Tag filter | `role="radiogroup"` no container, `role="radio"` + `aria-checked` em cada toggle |
| Tag filter "Todos" | `role="radio"` + `aria-checked` quando nenhum filtro de tag ativo |
| AlertDialog | shadcn AlertDialog com focus trap, Escape fecha, Tab navega entre botoes |
| Empty state CTA | `<Link>` semantico para `/leads/new`, focavel via Tab |
| Back button | `aria-label="Voltar"`, semantico `<button>` |
| Loading skeleton | `aria-busy="true"` no container da lista durante loading |
| List container | `aria-label="Lista de leads"` |
| Lead counter | `aria-live="polite"` para atualizar contagem ao filtrar |
| Focus order (lista) | Tab: TagFilter toggles -> LeadCards -> FAB |
| Focus order (edicao) | Tab: Voltar -> campos do form -> Salvar -> Excluir |
| Touch targets | Tag filter toggles: `min-h-[44px]`. LeadCard: full-width tocavel. |
| Color contrast | Tags oklch calibradas para ratio >= 4.5:1 (verificar pos-implementacao) |
