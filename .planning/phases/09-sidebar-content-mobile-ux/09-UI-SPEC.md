## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between avatar and text in user menu |
| md | 16px | Default element spacing, sidebar group padding, sidebar content padding |
| lg | 24px | Section padding, sidebar header height internal padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Touch targets:** Nav items use `min-h-11` (44px / 2.75rem) per D-09. This is 44px, not a multiple of 4 but mandated by WCAG 2.5.5.
- **Sidebar header:** `h-14` (56px) already set in Phase 8 -- retained as-is.
- **Avatar gap:** `gap-3` (12px) between avatar and text block in `SidebarUserMenu` for visual balance at `size-8` (32px) avatar.

## Interaction Contract

### 1. Mobile Drawer Auto-Close (MOBILE-02)

| Property | Value |
|----------|-------|
| Trigger | Any route change detected by `usePathname()` |
| Behavior | `useEffect` calls `setOpenMobile(false)` from `useSidebar()` hook |
| Timing | Immediate -- no transition delay, no debounce |
| Scope | Mobile only (< 768px) -- `setOpenMobile` is no-op on desktop |
| Placement | Inside `AppSidebar` component, at hook level |

### 2. Touch Targets (TOUCH-01)

| Element | Min Height | Implementation |
|---------|-----------|----------------|
| Vendedor nav items (`SidebarMenuButton`) | 44px | `className="min-h-11"` |
| Admin nav items (`SidebarMenuButton`) | 44px | `className="min-h-11"` |
| Collapsible trigger ("Admin" group label) | Default shadcn | No change per D-09 |
| Logout button | Default shadcn | No change per D-09 |

### 3. Active State (TOUCH-04)

| Route | Active Item(s) | Logic |
|-------|---------------|-------|
| `/dashboard` | Dashboard | `pathname.startsWith("/dashboard")` |
| `/leads` | Leads | `pathname === "/leads"` (exact match to avoid conflict with `/leads/new`) |
| `/leads/new` | Novo Lead | `pathname.startsWith("/leads/new")` |
| `/leads/[id]` | Leads | `pathname.startsWith("/leads")` -- sub-routes highlight parent |
| `/admin/leads` | Admin > Leads | `pathname.startsWith("/admin/leads")` |
| `/admin/leads/[id]` | Admin > Leads | `pathname.startsWith("/admin/leads")` -- nested routes highlight parent |
| `/admin/users` | Admin > Usuarios | `pathname.startsWith("/admin/users")` |
| `/admin/stats` | Admin > Stats Globais | `pathname.startsWith("/admin/stats")` |

**Active state visual:** Handled by `SidebarMenuButton` `isActive` prop -- applies `data-active` attribute which triggers `bg-sidebar-accent text-sidebar-accent-foreground` (shadcn default) plus sidebar-primary text/icon color.

### 4. User Menu in Footer (LAYOUT-08, POLISH-02)

| Element | Spec |
|---------|------|
| Avatar | 32x32px (`size="default"`, which is `size-8`), Gravatar image with `d=404`, AvatarFallback with 2-letter initials |
| Name | `text-sm font-medium`, truncate with `truncate` class, max 1 line |
| Role | `text-xs text-muted-foreground`, plain text "Admin" or "Vendedor", truncate |
| Logout | `Button` variant `ghost`, size `icon`, `LogOut` Lucide icon, `aria-label="Sair"` |
| Layout | Horizontal flex: `[Avatar] [Name + Role stacked] [Logout button]` |
| Separator | `SidebarFooter` component provides top border automatically via sidebar border token |
| Padding | `px-2 py-2` on inner flex container (8px horizontal, 8px vertical) |

### 5. iOS Safari Sheet (MOBILE-04)

| Property | Value |
|----------|-------|
| Height strategy | `inset-y-0` (already in Sheet component -- `top: 0; bottom: 0`) |
| Viewport units | Not used -- `inset-y-0` is immune to `dvh`/`svh` bugs |
| Action required | None -- existing CSS is already safe |
| Fallback | If iOS issues surface during QA, add `h-svh` as conservative override |

**Source:** RESEARCH.md Pitfall 4, `packages/ui/src/components/sheet.tsx`

### 6. ModeToggle Placement (LAYOUT-08)

| Property | Value |
|----------|-------|
| Location | Not in SidebarFooter for Phase 9 |
| Rationale | CONTEXT.md D-03 specifies "avatar + nome + role badge + botao Sair" only. ModeToggle migration is a LAYOUT-08 sub-item that can be addressed as a small addition during Phase 11 (POLISH-04: dark mode audit). Phase 9 focuses on the user menu core. |
| Current state | ModeToggle exists in `mode-toggle.tsx` but is not rendered anywhere after Header removal in Phase 8 |

### Fallback Copy

| Data Point | Fallback | When |
|------------|----------|------|
| User name | `user.email.split("@")[0]` | `user_metadata.full_name` is null/empty |
| User name (ultimate fallback) | `"Usuario"` | Both `full_name` and `email` are null |
| User email | `""` (empty string) | `user.email` is null |
| User role | `"vendedor"` | `user_role` claim not found in JWT |
| Gravatar URL | AvatarFallback renders initials | Gravatar returns 404 (no avatar for email hash) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Touch targets 44px (WCAG 2.5.5) | `min-h-11` on all `SidebarMenuButton` nav items |
| Keyboard navigation | shadcn SidebarMenu uses semantic `<button>` + `<nav>` -- keyboard nav built-in |
| Screen reader: logout | `aria-label="Sair"` on icon-only logout button |
| Screen reader: avatar | `alt={userName}` on `AvatarImage` |
| Focus visible | `outline-ring/50` globally applied via `globals.css` base layer |
| Drawer open/close | Sheet overlay traps focus via Base UI Dialog primitive |
| Color contrast | All sidebar tokens use oklch values with sufficient contrast ratios (light: dark text on near-white bg; dark: light text on dark bg) |
