## Spacing Scale

Declared values (must be multiples of 4). The project uses Tailwind 4 spacing units directly; all values listed are the Tailwind scale equivalents.

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, inline padding |
| sm | 8px | `gap-2`, `p-2` | Compact element spacing (card content gaps, leaderboard entry gaps) |
| md | 16px | `gap-4`, `p-4` | Default element spacing — grid gaps, page horizontal padding mobile |
| lg | 24px | `gap-6`, `p-6` | Section padding — page content padding on `md:` breakpoint |
| xl | 32px | `gap-8`, `p-8` | Layout gaps between major sections |
| 2xl | 48px | `gap-12` | Major section breaks (not used in this phase) |
| 3xl | 64px | `gap-16` | Page-level spacing (not used in this phase) |

**Exceptions:**

- AppTopbar height: `h-14` (56px) — consistent with existing `(app)/layout.tsx` `<header className="flex h-14 ...">`. Source: CONTEXT.md Claude's Discretion + RESEARCH.md Pattern 6.
- Touch targets: minimum `44px` on all interactive elements in topbar and stat cards. Source: REQUIREMENTS.md TOUCH-01 (carried forward from Phase 9).
- `SidebarTrigger` horizontal offset: `-ml-1` (−4px) — shadcn canonical pattern for flush alignment. Source: RESEARCH.md Pattern 1.
- Stat card grid gap: `gap-4` (16px) for all viewport sizes — matches `admin/stats/stats-panel.tsx` established pattern.

## Color

The project uses semantic CSS variables from `globals.css` (oklch color space). No hardcoded hex or oklch values in component className props — always use semantic tokens or `ChartConfig.theme`.

| Role | CSS Variable (light) | CSS Variable (dark) | Usage |
|------|---------------------|---------------------|-------|
| Dominant (60%) | `--background: oklch(1 0 0)` | `--background: oklch(0.145 0 0)` | Page background, SidebarInset background |
| Secondary (30%) | `--card: oklch(1 0 0)` / `--sidebar: oklch(0.985 0 0)` | `--card: oklch(0.205 0 0)` / `--sidebar: oklch(0.205 0 0)` | Stat cards, leaderboard cards, sidebar panel |
| Accent (10%) | `--primary: oklch(0.511 0.096 186.391)` (teal) | `--primary: oklch(0.437 0.078 188.216)` | Primary actions, current-user leaderboard border, breadcrumb active link |
| Destructive | `--destructive: oklch(0.577 0.245 27.325)` | `--destructive: oklch(0.704 0.191 22.216)` | Destructive actions only (not used in Phase 11) |
| Muted | `--muted-foreground: oklch(0.556 0 0)` | `--muted-foreground: oklch(0.708 0 0)` | Stat card labels, breadcrumb intermediate segments, leaderboard rank number |
| Border | `--border: oklch(0.922 0 0)` | `--border: oklch(1 0 0 / 10%)` | AppTopbar bottom border, card borders |

**Accent reserved for:**
- `SidebarTrigger` active/focus ring
- `BreadcrumbLink` hover state (via `text-foreground` on hover — shadcn default)
- Current-user `LeaderboardEntry` card border (`border-primary`)
- Primary `Button` variant (if used for destructive confirmation in future phases)

**Chart colors — Impeccable rule:** Chart `fill` props must reference `var(--color-key)` from `ChartConfig`, never inline oklch values. Use `ChartConfig.theme` with light/dark variants. Source: RESEARCH.md Pattern 5, Pitfall 3.

**Chart palette (theme-aware):**

**Dark mode rule:** Never add manual `dark:` overrides for semantic token colors. Use `dark:` only for oklch chart colors on StatCard label spans (existing codebase pattern in `personal-dashboard.tsx` — carry forward as-is until ChartConfig migration). Source: shadcn skill styling.md.

## Interaction Contract

### AppTopbar (POLISH-01)

**Visible on:** All authenticated pages inside `(app)/layout.tsx` — replaces `<header className="md:hidden">`.

**Layout:** `h-14` (56px height), `flex items-center gap-2 border-b px-4`, `shrink-0`.

**SidebarTrigger visibility:** `className="-ml-1 md:hidden"` — hidden on desktop (`md:` = 768px+). On desktop, sidebar is always expanded (Phase 8 D-07/D-09). Use CSS `md:hidden`, never `useIsMobile()` conditional rendering (avoids hydration mismatch). Source: CONTEXT.md D-02, RESEARCH.md Pitfall 5.

**Breadcrumb logic:**

| Route | Breadcrumb rendering |
|-------|---------------------|
| `/dashboard` | `BreadcrumbPage`: "Dashboard" |
| `/leads` | `BreadcrumbPage`: "Leads" |
| `/leads/new` | `BreadcrumbLink`: "Leads" + separator + `BreadcrumbPage`: "Novo Lead" |
| `/leads/[id]` | `BreadcrumbLink`: "Leads" + separator + `BreadcrumbPage`: "Detalhe" |
| `/admin/leads` | `BreadcrumbLink`: "Admin" + separator + `BreadcrumbPage`: "Leads" |
| `/admin/users` | `BreadcrumbLink`: "Admin" + separator + `BreadcrumbPage`: "Usuarios" |
| `/admin/stats` | `BreadcrumbLink`: "Admin" + separator + `BreadcrumbPage`: "Estatisticas" |

Dynamic segment detection: segment not in `ROUTE_LABELS` map AND matches `/^[0-9a-f-]{8,}$/` → render as "Detalhe". Source: RESEARCH.md Pattern 2, Pitfall 4.

`ROUTE_LABELS` map (exhaustive):
```
dashboard → "Dashboard"
leads → "Leads"
new → "Novo Lead"
admin → "Admin"
users → "Usuarios"
stats → "Estatisticas"
```

**Client boundary:** `AppTopbar` must be `"use client"` in a separate file — `(app)/layout.tsx` is a Server Component (`async` + `getUser()`). Source: RESEARCH.md Pitfall 2.

### Dashboard Stat Cards Grid (RESP-04)

**Stat cards in `personal-dashboard.tsx`:** Two separate responsive grids.

- Row 1 (4 cards): `grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-4` — Total de Leads, Leads Hoje, Quentes, Mornos.
- Row 2 (2 cards): `grid grid-cols-1 gap-4 sm:grid-cols-2` — Frios, Seu Score.

**Remove from `dashboard/page.tsx`:** `max-w-[480px]` constraint on the wrapper div. Replace with `w-full` only. Source: CONTEXT.md D-05, RESEARCH.md Pattern 3.

**Skeleton loading states** must be updated to match the new grid layout (2 skeletons in row 1, 2 skeletons in row 2 at `sm:grid-cols-2`, then grow to `lg:grid-cols-4`).

**Existing `space-y-4` in `personal-dashboard.tsx`:** Replace with `flex flex-col gap-4` during the Impeccable polish pass. Source: shadcn skill SKILL.md "No `space-y-*`".

### Chart Resize After Sidebar Toggle (RESP-05)

**Primary fix:** Remove `max-w-[480px]` from `dashboard/page.tsx`. This is the root cause — `ResponsiveContainer` observes its container, which was artificially constrained.

**Fallback (only if resize still incorrect after removing max-width):** Add `key={String(open)}` from `useSidebar()` to `ChartContainer`. The `open` boolean from `useSidebar()` changes on sidebar toggle and forces re-mount.

**Caution:** Key reset re-runs entry animations and loses tooltip hover state. Prefer CSS-only fix. Source: RESEARCH.md Pattern 4, Pitfall 1.

**`ChartContainer` className pattern:** `className="h-[120px] w-full"` — never hardcode width, never omit `w-full`. Source: CONTEXT.md D-07, RESEARCH.md Pattern 4.

### Leaderboard at 320px (POLISH-03)

**No layout change.** `LeaderboardEntry` remains vertical card layout. Decision D-08 locked this. Validation only: confirm no horizontal overflow at 320px viewport.

**Current layout confirmed correct:** `CardContent` with `flex items-start gap-3` — rank badge (`min-w-6 text-xs`), name+stats (`flex-1`). Width is constrained by card which fills container. Source: leaderboard-entry.tsx direct inspection.

### Dark Mode Audit (POLISH-04)

**All 7 authenticated routes** must be verified in both light and dark mode:
- `/dashboard`, `/leads`, `/leads/new`, `/leads/[id]`, `/admin/leads`, `/admin/users`, `/admin/stats`

**Known issues to fix during polish:**
1. `personal-dashboard.tsx` `chartConfig` uses `color: "oklch(...)"` (hardcoded). Migrate to `theme: { light: "...", dark: "..." }` pattern. Source: RESEARCH.md Pattern 5, Pitfall 3.
2. `chartData` array in `personal-dashboard.tsx` uses inline `fill: "oklch(...)"`. Change to `fill: "var(--color-quente)"` etc. Source: RESEARCH.md Pattern 5.
3. All `space-y-*` occurrences in polished files → `flex flex-col gap-*`.

**Rule:** No hardcoded color values in className. No `dark:` manual overrides for semantic token colors. Exception: the existing `dark:[&_span:first-child]:text-[oklch(...)]` pattern on StatCard `className` props is acceptable as-is — it is a targeted override for oklch label colors not expressible via ChartConfig. Source: RESEARCH.md Pitfall 3.

### Impeccable Polish — All Pages (POLISH-05)

**Skill sequence (applied globally by skill, not page by page):**

1. **arrange** — Audit all layout/spacing: replace `space-y-*` with `flex flex-col gap-*`, confirm grid gaps are consistent (`gap-4` standard).
2. **adapt** — Confirm all 7 routes render at 320px, 640px, 1024px without horizontal overflow.
3. **polish** — Micro-details: consistent `border-b` separators, `shrink-0` on fixed-height elements, `rounded-*` consistency.
4. **typeset** — Confirm heading hierarchy, `text-muted-foreground` on all secondary labels, `font-semibold` on all values, no raw color classes on text.

## Anti-Patterns (do not implement)

- `space-y-*` or `space-x-*` — use `flex flex-col gap-*` (shadcn rule)
- `w-10 h-10` — use `size-10` when dimensions are equal (shadcn rule)
- Inline oklch values in Recharts `fill` prop — use `var(--color-key)` via ChartConfig
- `useIsMobile()` for conditional rendering of SidebarTrigger — use CSS `md:hidden`
- Nested `SidebarProvider` — only one exists in `(app)/layout.tsx`
- `usePathname()` in a Server Component — `AppTopbar` must be `"use client"`
- Manual `dark:` color overrides for semantic token colors — use CSS variables
- `max-w-[480px]` on dashboard page container — remove entirely
