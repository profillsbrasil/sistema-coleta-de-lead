### Semantic State Colors for Sync Indicator

These map directly to the 5 states defined in CONTEXT.md D-06. Use Tailwind color utilities (not accent/primary variables) so the indicator communicates status independently of brand color.

| State | Precedence | Icon | Color Token | Tailwind Class | Rationale |
|-------|------------|------|-------------|----------------|-----------|
| Offline | 1 (highest) | `WifiOff` (Lucide) | --destructive | `text-destructive` | Matches destructive semantic — clear signal that sync is not possible |
| Syncing | 2 | `RefreshCw` (Lucide) | --primary | `text-primary animate-spin` | Active operation uses primary teal; spin via tw-animate-css `animate-spin` already in project |
| Error | 3 | `AlertTriangle` (Lucide) | amber-500 | `text-amber-500` | Warning, not destructive — error happened but app is functional |
| Pending | 4 | `CloudUpload` (Lucide) | --muted-foreground | `text-muted-foreground` | Neutral holding state; badge shows count |
| Synced | 5 (lowest) | `CloudCheck` (Lucide) | emerald-500 | `text-emerald-500` | Positive confirmation, distinct from primary brand teal |

**Icon size:** `size-4` (16px) — consistent with existing Lucide icons in the sidebar footer row.

**Badge (Pending state only):**
- Renders only when `pendingCount > 0`
- Diameter: 16px minimum (h-4 w-4 or min-w-4)
- Font: 10px / weight 600 / `tabular-nums`
- Color: bg-amber-500 text-white (warm, distinct from status icon itself)
- Position: absolute top-right overlay on the icon (`-top-1 -right-1`)
- Max display: 99+ if pendingCount > 99

## Interaction & Animation Contract

### Spin animation (Syncing state)

- Class: `animate-spin` from `tw-animate-css` (already in project via `@import "tw-animate-css"`)
- Duration: 1000ms (default `animate-spin` — 1 full rotation/second)
- Easing: linear (standard for continuous spinner)
- No entrance/exit animation on the spin itself — state change is immediate

### State transitions

- State changes are reflected immediately on next render (React context update)
- No transition animation between states — icon swaps instantly
- The icon must NOT flash/flicker when syncing completes: set `isSyncing = false` only after `lastSync` and `lastError` are also updated in the same state batch (single `setState` call in Provider)

### Tooltip

- Trigger: hover (desktop) — tooltip opens on hover with `delay={500}` (Base UI TooltipProvider default is 0; set 500ms delay to avoid accidental trigger)
- Mobile: tooltip is NOT the primary disclosure mechanism on mobile (no hover). The icon alone must communicate state without tooltip. This is explicitly noted in CONTEXT.md specifics: "o ícone sozinho deve ser compreensível mesmo sem hover (campo com luz solar, sem hover em mobile)."
- Tooltip placement: `side="top"` (icon is in footer — tooltip opens upward)
- `aria-label` on the trigger button must match tooltip text exactly so screen readers get the same information without hover

### Accessibility

- The trigger `<button>` must carry `aria-label` equal to the tooltip text for the current state
- `aria-live="polite"` on a visually-hidden span that announces state changes to screen readers (separate from tooltip)
- Icon is decorative (`aria-hidden="true"`) — meaning is carried by `aria-label` on the button

**Error state:** "Erro no ultimo sync" — descriptive, no action copy because retry is automatic (CONTEXT.md D-12). No "tentar novamente" button in this phase.

## Provider Architecture Contract

This section records the state contract so planner tasks can reference it directly.

### SyncStatusContext shape

```typescript
interface SyncStatus {
  isOnline: boolean;       // from ConnectivityDetector.isOnline
  isSyncing: boolean;      // set by engine callbacks
  pendingCount: number;    // from useLiveQuery(() => db.syncQueue.count())
  lastSync: string | null; // ISO timestamp, persisted in localStorage.lastSyncTimestamp
  lastError: string | null; // set only after maxRetries exhausted; cleared on next successful sync
}
```

**Source:** CONTEXT.md D-02

### Precedence rule (for SyncStatusIcon rendering logic)

```
if (!isOnline)               → Offline
else if (isSyncing)          → Syncing
else if (lastError !== null) → Error
else if (pendingCount > 0)   → Pending
else                         → Synced
```

**Source:** CONTEXT.md D-08

### localStorage key

- `lastSyncTimestamp` — existing key, already in use. No new keys introduced.
- **Source:** CONTEXT.md D-10
