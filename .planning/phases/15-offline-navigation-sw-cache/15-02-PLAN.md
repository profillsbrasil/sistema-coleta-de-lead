<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Protocolo de validacao manual — Chrome DevTools</name>
  <what-built>
    Service Worker completo em apps/web/public/sw.js com:
    - Pre-cache de /dashboard, /leads, /leads/new, /admin/*, /offline no install
    - NetworkFirst para RSC payloads (header RSC: 1) com URL normalizada (sem ?_rsc=)
    - CacheFirst para _next/static/** e fonts
    - setCatchHandler servindo /offline para rotas nao cacheadas offline
    - ServiceWorkerRegistrar integrado em Providers via workbox-window
    - Header Service-Worker-Allowed: / em next.config.ts
    - skipWaiting + clientsClaim para ativacao imediata
  </what-built>
  <how-to-verify>
    Execute o protocolo abaixo em ordem. Documente o resultado de cada passo (OK / FALHOU).

    ---

    ## PRE-REQUISITO

    - Iniciar servidor: `bun run dev:web` (porta 3001)
    - Abrir Chrome em http://localhost:3001
    - Fazer login com uma conta de teste
    - Abrir DevTools (F12)

    ---

    ## BLOCO 1: Registro do Service Worker (SW-02)

    **Passo 1.1 — Verificar registro:**
    - DevTools > Application > Service Workers
    - Esperado: entrada para `http://localhost:3001/sw.js` com status "activated and is running"
    - Scope deve ser `http://localhost:3001/`
    - Se status for "waiting to activate": clicar "skipWaiting" ou fechar e reabrir a aba

    **Passo 1.2 — Verificar ausencia de PWA:**
    - DevTools > Application > Manifest
    - Esperado: "No manifest detected" ou aba vazia
    - Nao deve aparecer nenhum manifest.json vinculado ao app

    **Passo 1.3 — Verificar ausencia de install prompt:**
    - Nenhum banner de "Instalar app" deve aparecer
    - Nenhum icone de instalacao na barra de endereco do Chrome

    ---

    ## BLOCO 2: Pre-cache no Install (SW-01, criterio 1)

    **Passo 2.1 — Verificar Cache Storage apos registro:**
    - DevTools > Application > Cache Storage
    - Deve existir cache `workbox-precache-v2-http://localhost:3001/`
    - Deve conter entradas para: `/dashboard`, `/leads`, `/leads/new`, `/admin/leads`, `/admin/users`, `/offline`

    **Passo 2.2 — Navegar entre rotas:**
    - Clicar em cada item do menu (Dashboard, Leads, etc.)
    - Apos navegar, verificar Cache Storage > `rsc-payloads-v1`
    - Esperado: entradas com URLs sem `?_rsc=` (ex: `http://localhost:3001/dashboard`, nao `http://localhost:3001/dashboard?_rsc=abc`)

    **Passo 2.3 — Verificar assets estaticos:**
    - Cache Storage > `static-assets-v1`
    - Esperado: entradas para arquivos em `/_next/static/`

    ---

    ## BLOCO 3: Navegacao Offline (SW-01, criterio 1 e 3)

    **Passo 3.1 — Ir para modo offline:**
    - DevTools > Network > throttling dropdown > "Offline"
    - Verificar que o indicador de sync no app mostra "offline" (SyncStatusIcon da Phase 13)

    **Passo 3.2 — Navegar entre rotas cacheadas:**
    - Clicar em Dashboard no menu
    - Esperado: pagina carrega normalmente, sem "Failed to fetch RSC payload"
    - Clicar em Leads no menu
    - Esperado: pagina carrega normalmente
    - Clicar em "Novo Lead" (rota /leads/new)
    - Esperado: formulario de captura disponivel

    **Passo 3.3 — Verificar que o formulario de captura funciona offline:**
    - Na pagina /leads/new offline
    - Preencher os campos e tentar salvar
    - Esperado: lead salvo no IndexedDB (Dexie), pendingCount incrementado no SyncStatusIcon
    - O formulario NAO deve quebrar porque o SW nao intercepta chamadas tRPC (D-02)

    **Passo 3.4 — Verificar fallback /offline:**
    - Ainda em modo offline, navegar para uma rota nao pre-cacheada
    - Tentar: digitar na barra um path nao cacheado (ex: `/leads/123` se nao visitado antes)
    - Esperado: pagina /offline e exibida com "Sem conexao" e link para /dashboard
    - Se /leads/[id] for pre-cacheado, testar com outra rota que nao esteja na lista de precache

    ---

    ## BLOCO 4: Volta ao Online e Atualizacao Transparente (SW-01, criterio 4)

    **Passo 4.1 — Voltar ao online:**
    - DevTools > Network > throttling dropdown > "No throttling"
    - SyncStatusIcon deve transicionar para "syncing" e depois "synced"
    - Nenhum reload manual necessario

    **Passo 4.2 — Verificar atualizacao de cache:**
    - Navegar entre rotas
    - Cache Storage > `rsc-payloads-v1` deve ter entradas atualizadas (timestamps recentes)
    - O SW atualiza cache silenciosamente — sem notificacao ao usuario

    ---

    ## BLOCO 5: Console — Ausencia de Erros

    **Passo 5.1 — Verificar console:**
    - DevTools > Console
    - Nao deve haver erros de registro do SW
    - Nao deve haver DOMException relacionada a scope
    - Nao deve haver erros de workbox-sw CDN
    - Nao deve haver console.log statements do codigo do app (apenas mensagens do Workbox sao aceitaveis em dev)

    ---

    ## RESULTADO ESPERADO

    Todos os 4 success criteria do ROADMAP devem ser verdadeiros:
    1. [ ] Navegacao offline entre rotas autenticadas sem RSC error
    2. [ ] Nenhum manifest, nenhum install prompt, nenhuma funcionalidade PWA
    3. [ ] Formulario de captura funciona offline sem interrupcao
    4. [ ] Cache atualizado transparentemente apos reconectar

    ---

    Se qualquer passo falhar, documentar o comportamento observado e o erro exato.
    Nao continuar para Safari se Chrome apresentar falhas criticas — resolver primeiro.
