## Interaction Contract

### Fluxo de captura (3 toques)

1. Vendedor toca no FAB (toque 1) -> navega para `/leads/new`
2. Vendedor preenche nome + telefone/email (input, nao contabilizado como toque)
3. Vendedor toca "Salvar Lead" (toque 2) -> lead salvo no Dexie

O terceiro toque (tag) nao e necessario porque "Morno" e default. Se o vendedor quiser trocar, e toque 2 (tag) + toque 3 (salvar).

### QR Scan

1. Vendedor toca icone QR ao lado do campo telefone
2. Overlay abre com camera ativa (getUserMedia)
3. Scan detecta QR code -> parseia wa.me URL -> extrai numero
4. Overlay fecha automaticamente, campo telefone preenchido
5. Toast "Telefone detectado"
6. Se QR invalido ou nao wa.me: toast de erro, overlay permanece

### Foto de cartao

1. Vendedor toca botao camera
2. Camera nativa abre (`<input type="file" accept="image/*" capture="environment">`)
3. Foto capturada -> comprimida via canvas (max 1280px, JPEG 0.7) -> base64
4. Preview 80x80px aparece no form
5. Botao "X" no canto do preview para remover
6. Foto salva no Dexie como Blob junto ao lead

### Validacao

- Validacao no submit (nao em tempo real para evitar friccao durante coleta rapida)
- Campos obrigatorios: nome, pelo menos um de telefone/email, tag (sempre tem default)
- Erros inline: border vermelha + mensagem abaixo do campo
- Focus automatico no primeiro campo com erro
- Schema via Zod no client (reutilizar schema de validacao)

### Navegacao pos-salvar

- Apos salvar com sucesso: `router.back()` (volta para onde veio — dashboard ou lista)
- Se nao houver history: redirect para `/dashboard`

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Form labels | Cada campo tem `<Label htmlFor={id}>` associado |
| Tag selector | `role="radiogroup"` no container, `role="radio"` + `aria-checked` em cada tag |
| QR button | `aria-label="Escanear QR Code do WhatsApp"` |
| Camera button | `aria-label="Tirar foto do cartao de visita"` |
| FAB | `aria-label="Adicionar novo lead"` |
| Photo preview | `alt="Preview da foto capturada"` no `<img>` |
| Remove photo | `aria-label="Remover foto"` no botao X |
| Back button | `aria-label="Voltar"` |
| Error messages | `aria-describedby` linking input ao erro, `role="alert"` na mensagem |
| Loading state | `aria-busy="true"` no botao Salvar durante submit |
| Focus order | Tab: Nome -> Telefone -> QR btn -> Email -> Tags -> Camera btn -> Mais detalhes -> (campos opcionais) -> Salvar |
| Touch targets | Todos os botoes minimo 44x44px de area tocavel. Tags: `min-h-[44px]`. QR/Camera btns: size `icon` (40px) + padding |
| Color contrast | Tags usam oklch calibrado para ratio >= 4.5:1 texto sobre fundo (verificar pos-implementacao) |
