- Query vendors list (only when admin):

});

- The effective userId for PersonalDashboard: `selectedVendor || userId`

- Render vendor selector inline with TabsList, to the right of tabs (per D-03):
```tsx
<Tabs defaultValue="dashboard">
  <div className="flex items-center justify-between gap-4">
    <TabsList>
      <TabsTrigger value="dashboard">Meu Dashboard</TabsTrigger>
      <TabsTrigger value="leaderboard">Leaderboard</TabsTrigger>
    </TabsList>
    {isAdmin && vendorsQuery.data ? (
      <Select
        onValueChange={(v) => setSelectedVendor(v === "self" ? undefined : v)}
        value={selectedVendor ?? "self"}
      >
        <SelectTrigger className="w-[200px]">
          <SelectValue placeholder="Selecionar vendedor" />
        </SelectTrigger>
        <SelectContent>
          <SelectItem value="self">Meu Dashboard</SelectItem>
          {vendorsQuery.data.map((v) => (
            <SelectItem key={v.userId} value={v.userId}>{v.name}</SelectItem>
          ))}
        </SelectContent>
      </Select>
    ) : null}
  </div>
  <TabsContent className="mt-4" value="dashboard">
    <PersonalDashboard userId={selectedVendor || userId} />
  </TabsContent>
  <TabsContent className="mt-4" value="leaderboard">
    <LeaderboardTab userId={userId} />
  </TabsContent>
</Tabs>
```

- Import Select components from `@dashboard-leads-profills/ui/components/select`
- Import trpc from `@/utils/trpc`

- NOTE: PersonalDashboard uses `useLiveQuery(() => getPersonalStats(userId))` which queries Dexie. When admin selects another vendor, Dexie won't have that vendor's data locally. This means the vendor selector on the /dashboard page will only show data that exists in the local Dexie database. For the admin's own dashboard it works fine. For other vendors, the data would be empty unless synced locally. This is a known limitation — the proper admin stats view is at /admin/stats (D-13). The vendor selector on /dashboard per D-03 provides a quick glance but relies on local data. Document this as a known behavior. An alternative: add a tRPC-backed PersonalDashboard variant for admin, but this would be over-engineering for D-03 since /admin/stats already covers the full admin stats use case.
