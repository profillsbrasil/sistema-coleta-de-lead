1. **packages/api/src/routers/admin/stats.ts** — Create admin stats router using `adminProcedure`. Procedures (per D-13, D-14, D-15, ADMN-06):

- `getGlobalStats`: input `{ userId?: z.uuid(), tag?: z.enum(["quente", "morno", "frio"]), segment?: z.string(), startDate?: z.string(), endDate?: z.string() }`. Build dynamic WHERE conditions array: always include `isNull(leads.deletedAt)`. Add conditions for each optional filter. Query:
```sql
SELECT
  count(*)::int AS "total",
  count(*) FILTER (WHERE interest_tag = 'quente')::int AS "quente",
  count(*) FILTER (WHERE interest_tag = 'morno')::int AS "morno",
  count(*) FILTER (WHERE interest_tag = 'frio')::int AS "frio",
  COALESCE(sum(CASE WHEN interest_tag = 'quente' THEN 3 WHEN interest_tag = 'morno' THEN 2 ELSE 1 END), 0)::int AS "score"
FROM leads WHERE {conditions}
```
Also count today's leads (same conditions + `created_at >= today 00:00 in America/Sao_Paulo`) and count active vendors (distinct userId from leads where deletedAt IS NULL).

- `getTimeline`: input same filters as getGlobalStats. Query:
```sql
SELECT DATE(created_at AT TIME ZONE 'America/Sao_Paulo') as "date", COUNT(*)::int as "count"
FROM leads WHERE {conditions}
GROUP BY DATE(created_at AT TIME ZONE 'America/Sao_Paulo')
ORDER BY "date" ASC
```
Return `Array<{ date: string, count: number }>`.

- `getRanking`: input same filters. Query similar to leaderboard getRanking but with filter support and with names resolved via raw SQL JOIN on `auth.users`. Return `Array<{ userId, name, totalLeads, score }>` ordered by score DESC.

- `getDistinctSegments`: no input. Query `SELECT DISTINCT segment FROM leads WHERE segment IS NOT NULL AND deleted_at IS NULL ORDER BY segment`. Return `string[]`. Used to populate the segment filter dropdown.
