2. **apps/web/src/app/admin/users/users-panel.tsx** — Client Component ("use client"). Per UI-SPEC:

**State:**
- `search: string` — search input text (debounced for query)
- `page: number` — current page (starts at 1)
- `editingUserId: string | null` — user being role-edited (inline)
- `editingRole: "admin" | "vendedor"` — new role selection
- `deactivatingUser: { id: string, name: string } | null` — for deactivate AlertDialog
- `reactivatingUser: { id: string, name: string } | null` — for reactivate AlertDialog

**Queries:**
- `trpc.admin.users.list.useQuery({ search: debouncedSearch, page, perPage: 20 })`

**Mutations:**
- `trpc.admin.users.updateRole.useMutation({ onSuccess: invalidate list })`
- `trpc.admin.users.deactivate.useMutation({ onSuccess: invalidate list, toast success })`
- `trpc.admin.users.reactivate.useMutation({ onSuccess: invalidate list, toast success })`

**Get current user ID** to prevent self-deactivation:
```typescript
const [currentUserId, setCurrentUserId] = useState<string | null>(null);
useEffect(() => {
  const supabase = createClient();
  supabase.auth.getUser().then(({ data }) => setCurrentUserId(data.user?.id ?? null));
}, []);
```

**Layout:**

a) Page heading: `<h1 className="font-semibold text-xl mb-6">Gerenciar Usuarios</h1>`

b) Search input: Input with Search icon prefix (lucide-react). `placeholder="Buscar por nome ou email"`. On change, update search state. Use `setTimeout` for 300ms debounce.

c) Pagination info: `<p className="text-muted-foreground text-sm text-right mb-2">Mostrando {start}-{end} de {total}</p>`

d) Table columns per UI-SPEC:
   - Nome: user name from metadata
   - Email: user email
   - Role: Badge component.
     - Admin: `<Badge variant="default">` with Shield icon (lucide) + "Admin" text
     - Vendedor: `<Badge variant="secondary">Vendedor</Badge>` (muted)
   - Status: Badge component.
     - Active: `<Badge className="bg-green-100 text-green-800 dark:bg-green-900 dark:text-green-200">Ativo</Badge>`
     - Banned: `<Badge variant="destructive">Desativado</Badge>`
   - Leads: number from `leadCount`
   - Acoes: icon buttons with Tooltip:
     - Pencil (edit role): `aria-label="Editar usuario"`. On click, set `editingUserId` to this user's id and `editingRole` to current role. Show inline Select to pick new role. On Select change, call updateRole mutation immediately.
     - Ban icon (deactivate): `aria-label="Desativar usuario"`. Only show if user is not banned AND is not current user. On click, open AlertDialog.
     - CheckCircle icon (reactivate): `aria-label="Reativar usuario"`. Only show if user IS banned. On click, open AlertDialog.

e) Inline role editing: When `editingUserId` matches a row, replace the Role Badge cell with a Select component:
```tsx
<Select
  defaultValue={editingRole}
  onValueChange={(value) => {

    setEditingUserId(null);
  }}

  <SelectTrigger className="w-[120px]">
    <SelectValue />
  </SelectTrigger>

```
