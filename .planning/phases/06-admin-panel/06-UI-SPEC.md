## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, table cell padding |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, sidebar item spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions: Sidebar width fixed at 256px (collapsed: 48px on mobile via Sheet). Touch targets minimum 44px for mobile sidebar items.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Display (28px) used for stats numbers in StatCard and global stats (existing pattern from Phase 5).
- Label (12px) used for table column headers, filter labels, muted-foreground metadata. Uses regular weight (400) for visual hierarchy without introducing a third weight.
- Body (14px) is the default for table cells, form fields, sidebar items, and sidebar header text.
- Heading (20px) used for page titles ("Leads", "Usuarios", "Stats Globais") and CardTitle.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` oklch(1 0 0) / dark: oklch(0.145 0 0) | Page background, main content area |
| Secondary (30%) | `var(--card)` oklch(1 0 0) / dark: oklch(0.205 0 0) | Cards, sidebar background (`var(--sidebar)`), table rows, filter bar |
| Accent (10%) | `var(--primary)` oklch(0.511 0.096 186.391) / dark: oklch(0.437 0.078 188.216) | Primary CTA buttons, active sidebar item, selected vendor indicator |
| Destructive | `var(--destructive)` oklch(0.577 0.245 27.325) / dark: oklch(0.704 0.191 22.216) | Delete lead button, deactivate user button, destructive confirmation dialog |

Accent reserved for:
- Primary action buttons ("Salvar usuario", "Aplicar Filtros")
- Active/selected sidebar navigation item (via `--sidebar-primary`)
- Selected vendor badge in vendor selector
- Chart primary bar (total leads)

Semantic tag colors (existing, reuse from Phase 5):
- Quente: `oklch(0.45 0.18 17)` / dark: `oklch(0.85 0.12 17)`
- Morno: `oklch(0.5 0.13 85)` / dark: `oklch(0.85 0.1 85)`
- Frio: `oklch(0.45 0.15 240)` / dark: `oklch(0.85 0.1 240)`

Chart colors use existing CSS variables: `--chart-1` through `--chart-5` (teal scale).

## Layout Contract

### Admin Sidebar

- Width: 256px (desktop), Sheet overlay (mobile via SidebarTrigger)
- Background: `var(--sidebar)` — oklch(0.985 0 0) light / oklch(0.205 0 0) dark
- Items ordered top to bottom:
  1. "Leads" (ClipboardList icon) — `/admin/leads`
  2. "Usuarios" (Users icon) — `/admin/users`
  3. "Stats Globais" (LayoutDashboard icon) — `/admin/stats`
- Active item uses `--sidebar-primary` for background highlight
- Sidebar header: text "Admin" in 14px regular (400) weight — uses Body scale size
- Separator below header

### Admin Layout Structure

```
+------------------+--------------------------------------+
| Sidebar (256px)  | Main content (flex-1)                |
|                  |  +----------------------------------+ |
| [Admin]          |  | Page heading (20px semibold)     | |
| -----------      |  +----------------------------------+ |
| Leads            |  | Filter bar (if applicable)       | |
| Usuarios         |  +----------------------------------+ |
| Stats Globais    |  | Content (table / stats / form)   | |
|                  |  +----------------------------------+ |
|                  |  | Pagination (bottom)              | |
+------------------+--------------------------------------+
```

Main content area: `padding: 24px` (lg token). Min height: `calc(100svh - 49px)` (header height = 49px, existing pattern).

### Leads Page (`/admin/leads`)

- Top: Vendor selector (Select component) — "Selecionar vendedor" placeholder
- Below selector: When vendor selected, show vendor stats row (grid of 3 StatCards: Total, Score, Leads Hoje)
- Below stats: Table with leads for selected vendor
- Table columns: Nome | Telefone/Email | Tag | Segmento | Criado em | Acoes
- Tag column: Badge with oklch tag color (quente/morno/frio)
- Actions column: Pencil (edit) + Trash2 (delete) icon buttons, 32px touch targets, with Tooltip and aria-label
- Pagination: 20 items per page, bottom-aligned
- Empty state when no vendor selected: "Selecione um vendedor para ver seus leads."

### Users Page (`/admin/users`)

- Top: Search input (by name) with Search icon prefix
- Below: Table with users
- Table columns: Nome | Email | Role | Status | Leads | Acoes
- Role column: Badge — "Admin" (Shield icon, primary color) or "Vendedor" (muted)
- Status column: Badge — "Ativo" (green tint) or "Desativado" (destructive tint)
- Actions column: Pencil (edit role/name) + Ban/CheckCircle (deactivate/reactivate), with Tooltip and aria-label
- Pagination: 20 items per page

### Stats Page (`/admin/stats`)

- Top: Filter bar in a Card — Vendor select + Tag select + Segmento select + Date range picker + "Aplicar" button
- Segmento select: placeholder "Todos os segmentos", lists distinct segment values from leads data
- Date range presets: "Hoje", "Ultimos 7 dias", "Ultimos 30 dias", "Todo periodo"
- Below filters: Stats grid (4 StatCards in a row on desktop, 2x2 on mobile):
  - Total de Leads | Score Total | Leads Hoje | Vendedores Ativos
- Below stats: Two charts side by side (desktop), stacked (mobile):
  - Left: BarChart vertical — leads by tag (reuse Phase 5 pattern)
  - Right: LineChart — leads por dia (timeline), using `--chart-1` for the line color
- Below charts: Ranking table (reuse LeaderboardEntry or Table with rank/name/leads/score)

### Dashboard Vendor Selector (D-03)

- Position: Inline with existing TabsList, to the right of the tabs
- Only rendered when user role is admin
- Select component with "Todos" default + list of vendors by name
- When a vendor is selected, PersonalDashboard renders stats for that vendor instead of the current user

## Interaction States

### Loading States

| Context | Pattern |
|---------|---------|
| Table loading | 5 Skeleton rows matching table column structure |
| Stats cards loading | Grid of Skeleton cards matching layout (4 cards) |
| Chart loading | Single Skeleton rectangle at chart height (160px) |
| Vendor selector loading | Select disabled with "Carregando..." text |

### Error States

| Context | Pattern |
|---------|---------|
| Table fetch error | Empty component with error copy + "Tentar novamente" text button |
| Stats fetch error | Card with error copy, centered |
| User action error (delete/deactivate) | Sonner toast with error message |
| User action success | Sonner toast with success message |

### Confirmation Flows

| Action | Flow |
|--------|------|
| Delete lead | Click Trash2 -> AlertDialog with warning copy -> "Cancelar" / "Excluir" (destructive variant) |
| Deactivate user | Click Ban -> AlertDialog with warning copy -> "Cancelar" / "Desativar" (destructive variant) |
| Reactivate user | Click CheckCircle -> AlertDialog with confirmation copy -> "Cancelar" / "Reativar" (primary variant) |
| Edit lead | Click Pencil -> Navigate to /admin/leads/[id] with LeadForm (server-only save) |
| Edit user role | Click Pencil -> Inline Select in table row or modal with role selector + "Salvar usuario" |

## Responsive Behavior

| Breakpoint | Layout |
|------------|--------|
| < 768px (mobile) | Sidebar hidden, accessible via SidebarTrigger (hamburger). Stats grid 2x2. Charts stacked. Tables scroll horizontally. |
| >= 768px (tablet) | Sidebar visible, 256px. Stats grid 2x2. Charts stacked. |
| >= 1024px (desktop) | Sidebar visible, 256px. Stats grid 4-col. Charts side by side. Full table visible. |

---

## Pagination

- Strategy: Server-side pagination via tRPC (limit/offset)
- Page size: 20 items
- UI: shadcn Pagination component at bottom of table
- Display: "Mostrando {start}-{end} de {total}" text above table, right-aligned
- Rationale: Admin operates server-only (D-07), no Dexie. Infinite scroll inappropriate for admin data management; explicit pagination provides clear position context.
