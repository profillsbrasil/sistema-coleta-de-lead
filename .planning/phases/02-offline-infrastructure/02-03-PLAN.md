  <behavior>
    - startSync() initializes the engine and returns a cleanup function
    - syncCycle() performs push-then-pull:
      1. Push: read syncQueue ordered by timestamp, send via syncClient.sync.pushChanges.mutate. On success: delete acknowledged items from syncQueue, update serverId and syncStatus="synced" on created leads.
      2. Pull: read lastSyncTimestamp from localStorage, query syncClient.sync.pullChanges.query({ since }). For each server lead: if local lead exists with newer updatedAt, skip (will push next cycle). Otherwise, apply server data (server-wins per D-08), increment conflict counter. Save serverTimestamp to localStorage.
    - Conflict resolution: row-level server-wins based on updated_at (per D-08). Toast via sonner when conflicts occur (per D-09).
    - Mutex: isSyncing flag prevents concurrent sync cycles (pitfall 3 from research).
    - Error handling (per D-05, OFFL-06):
      - On 401: stop sync, do NOT clear syncQueue or Dexie data. Log warning.
      - On network error: retry with exponential backoff up to maxRetries.
      - On any failure: syncQueue and local leads are NEVER deleted.
    - tRPC vanilla client: separate instance using createTRPCClient with httpBatchLink, credentials: "include", url: "/api/trpc"
    - Connectivity integration: subscribe to connectivity detector, trigger syncCycle on online=true
  </behavior>
  <action>
    1. Create `apps/web/src/lib/sync/engine.ts` (per D-05, D-06, D-07, D-08, D-09):
       - Create tRPC vanilla client singleton (per D-07):
         ```
         const syncClient = createTRPCClient<AppRouter>({
           links: [httpBatchLink({
             url: "/api/trpc",
             fetch(url, options) { return fetch(url, { ...options, credentials: "include" }); },
           })],
         });
         ```
       - State: `let isSyncing = false;` (mutex per research pitfall 3)
       - `async function pushChanges()`:
         - Read all items from `db.syncQueue.orderBy("timestamp").toArray()`
         - If empty, return early
         - Call `syncClient.sync.pushChanges.mutate({ operations: pendingOps.map(...) })`
         - On success: `db.syncQueue.bulkDelete(result.acknowledged.map(a => a.queueId))` — only delete ACK'd items (per research: never clear before ACK)
         - For idMappings: `db.leads.update(mapping.localId, { serverId: mapping.serverId, syncStatus: "synced" })`
       - `async function pullChanges()`:
         - Read `lastSyncTimestamp` from localStorage (default "1970-01-01T00:00:00Z")
         - Call `syncClient.sync.pullChanges.query({ since: lastSync })`
         - For each server lead:
           - Get local lead by localId from Dexie
           - If local exists and local.updatedAt > server.updatedAt AND local.syncStatus === "pending": skip (local is newer, will push next cycle)
           - Otherwise: `db.leads.put({ ...serverLead mapped to Dexie Lead shape, syncStatus: "synced" })` — server wins (per D-08)
           - Count conflicts (local existed and was overwritten)
         - Save `serverTimestamp` to localStorage
         - If conflicts > 0: `toast.info("X leads atualizados pelo servidor")` (per D-09: discrete toast, not blocking)
       - `async function syncCycle()`:
         - If `isSyncing`, return (mutex)
         - Set `isSyncing = true`
         - try: await pushChanges(), await pullChanges() (per D-06: push first, then pull)
         - catch: if error is 401/UNAUTHORIZED: stop retrying, data preserved (per OFFL-06). If network error: throw for retry handler.
         - finally: `isSyncing = false`
       - `async function syncWithRetry()`:
         - Loop up to SYNC_CONFIG.maxRetries
         - try syncCycle(), return on success
         - On 401: break immediately (per OFFL-06)
         - On other error: await delay using getBackoffDelay(retryCount) (per D-05)
       - `export function startSync(): () => void`:
         - Create connectivity detector via createConnectivityDetector()
         - Subscribe to online changes: when online=true, call syncWithRetry()
         - Start connectivity detector
         - If already online, trigger initial sync
         - Return cleanup function that calls detector.stop()
  </action>
