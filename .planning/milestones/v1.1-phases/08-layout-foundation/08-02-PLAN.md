    **Part A: Create `apps/web/src/app/(app)/layout.tsx`**

    This is an async Server Component (per D-10, LAYOUT-07, Research Pattern 2).

    ```typescript
    import {
      SidebarProvider,
      SidebarInset,
      SidebarTrigger,
    } from "@dashboard-leads-profills/ui/components/sidebar";
    import { redirect } from "next/navigation";
    import AppSidebar from "@/components/app-sidebar";
    import { createClient } from "@/lib/supabase/server";

    export default async function AppLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      const supabase = await createClient();
      const {
        data: { user },
      } = await supabase.auth.getUser();

      if (!user) {
        redirect("/login");
      }

      const { data: claimsData } = await supabase.auth.getClaims();
      const userRole = (claimsData?.claims as Record<string, unknown>)?.user_role;
      const isAdmin = userRole === "admin";

      return (
        <SidebarProvider defaultOpen>
          <AppSidebar isAdmin={isAdmin} />
          <SidebarInset>
            <header className="flex h-14 items-center gap-2 border-b px-4 md:hidden">
              <SidebarTrigger />
            </header>
            <div className="flex-1 p-4 md:p-6">
              {children}
            </div>
          </SidebarInset>
        </SidebarProvider>
      );
    }
    ```

    Key details:
    - `SidebarProvider defaultOpen` — sidebar starts expanded, cookie persists state (per UI-SPEC Layout Contract)
    - `SidebarTrigger` only visible on mobile (`md:hidden`) — on desktop sidebar is always expanded because no SidebarTrigger or SidebarRail is rendered (per D-07, D-09, UI-SPEC SidebarTrigger Visibility)
    - `SidebarInset` renders a `<main>` tag and handles peer-data margins automatically
    - `<header>` for mobile top bar has `h-14` height, `border-b` bottom border, uses `gap-2` (not `space-x-2`)
    - Content area uses `flex-1 p-4 md:p-6` (per UI-SPEC Spacing Scale: 16px mobile, 24px desktop)
    - Auth guard pattern is IDENTICAL to current `admin/layout.tsx` — `getUser()` + `getClaims()` + `redirect("/login")`
    - `isAdmin` passed as prop to AppSidebar (per D-11)

    ```typescript
    import { redirect } from "next/navigation";
    import { createClient } from "@/lib/supabase/server";

    export default async function AdminLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      const supabase = await createClient();
      const {
        data: { user },
      } = await supabase.auth.getUser();

      if (!user) {
        redirect("/login");
      }

      const { data: claimsData } = await supabase.auth.getClaims();
      const userRole = (claimsData?.claims as Record<string, unknown>)?.user_role;

      if (userRole !== "admin") {
        redirect("/dashboard");
      }

      return <>{children}</>;
    }
    ```

    Key details:
    - This layout ONLY checks role — `userRole !== "admin"` redirects to `/dashboard`
    - NO SidebarProvider, NO AdminSidebar import, NO layout wrapper (just pass-through `{children}`)
