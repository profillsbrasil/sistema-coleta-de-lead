## Layout Contract

### Sidebar Shell Dimensions

| Property | Value | Source |
|----------|-------|--------|
| Desktop sidebar width | 16rem (256px) | `SIDEBAR_WIDTH` constant in sidebar.tsx |
| Mobile drawer width | 18rem (288px) | `SIDEBAR_WIDTH_MOBILE` constant in sidebar.tsx |
| Sidebar collapsible mode | `"offcanvas"` (default) | D-07, D-08, D-09 + Research Pattern 1 |
| Mobile breakpoint | 768px (md) | `useIsMobile` hook in packages/ui |
| Content area min-height | 100svh (via SidebarProvider's `min-h-svh`) | SidebarProvider internals |

### Structural Layout

```
Root layout.tsx:
  <html>
    <body>
      <Providers>
        {children}              # No grid wrapper, no Header
      </Providers>
    </body>
  </html>

(public)/layout.tsx:
  {children}                    # Pass-through, login has own centering

(app)/layout.tsx:
  <SidebarProvider defaultOpen>
    <AppSidebar isAdmin={isAdmin} />
    <SidebarInset>
      <header className="flex h-14 items-center gap-2 border-b px-4 md:hidden">
        <SidebarTrigger />
      </header>
      <div className="flex-1 p-4 md:p-6">
        {children}
      </div>
    </SidebarInset>
  </SidebarProvider>
```

### SidebarTrigger Visibility

| Viewport | Visible | Element |
|----------|---------|---------|
| Mobile (< 768px) | Yes | Hamburger icon in SidebarInset header bar |
| Desktop (>= 768px) | No | `md:hidden` hides the trigger; sidebar always expanded |

### SidebarRail

Do NOT render `SidebarRail`. Without it, desktop sidebar has no drag handle to collapse. Combined with no `SidebarTrigger` on desktop, the sidebar stays permanently expanded (D-07, D-09).

Source: CONTEXT.md D-07, D-08, D-09; RESEARCH.md Pattern 1.

## Interaction Contract

### Desktop (>= 768px)

| Interaction | Behavior |
|-------------|----------|
| Page load (authenticated) | Sidebar visible on left, content to the right. No animation. |
| Page load (unauthenticated) | Redirect to /login (server-side, no flash). |
| Click nav item | Navigate to route. Active state updates via pathname comparison. |
| Ctrl+B / Cmd+B | Accepted as harmless -- sidebar collapses but re-expands on next page load via `defaultOpen`. No override needed in Phase 8. |
| Admin section | Collapsible group, default open. Click group label toggles visibility. |

### Mobile (< 768px)

| Interaction | Behavior |
|-------------|----------|
| Page load | Sidebar hidden. Hamburger button (SidebarTrigger) visible in top header bar. |
| Tap hamburger | Sheet drawer opens from left edge, width 18rem (288px). |
| Tap nav item | Navigate to route. Drawer does NOT auto-close in Phase 8 (deferred to Phase 9 via MOBILE-02). |
| Tap outside drawer | Sheet closes (built-in shadcn Sheet behavior). |

### Transitions

| Element | Animation | Duration |
|---------|-----------|----------|
| Sheet drawer open/close | Slide from left | shadcn Sheet default (~200ms) |
| SidebarMenuButton hover | Background color change to sidebar-accent | Tailwind transition-colors default |
| Collapsible admin group | Height expand/collapse | shadcn Collapsible default |

Source: CONTEXT.md D-07, D-08, D-09; RESEARCH.md Pattern 1; Deferred items.
