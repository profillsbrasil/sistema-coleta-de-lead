## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, gap between badge icon and text |
| sm | 8px | Compact element spacing, card internal gaps between lines |
| md | 16px | Default card padding, form field gaps, card list gap |
| lg | 24px | Section padding (page content padding-inline on mobile) |
| xl | 32px | Layout gaps between major sections (header-to-content) |
| 2xl | 48px | Page top/bottom padding |
| 3xl | 64px | Empty state vertical centering margin |

Exceptions:
- **44px** minimum touch target on DropdownMenu trigger, FAB, and all interactive card elements (WCAG 2.5.5)
- **56px (h-14)** FAB size (existing `h-14 w-14` in `fab.tsx`) -- maintained for consistency
- **200px** rootMargin for IntersectionObserver sentinel (existing value in `lead-list.tsx`)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.43 (20px) |
| Label | 14px (`text-sm`) | 400 (regular) | 1.43 (20px) |
| Heading | 20px (`text-xl`) | 600 (semibold) | 1.4 (28px) |
| Caption | 12px (`text-xs`) | 400 (regular) | 1.33 (16px) |

Two weights only: 400 (regular) for body, labels, and captions; 600 (semibold) for headings and emphasis. Labels differentiate from body text via context and color (`text-muted-foreground` for secondary labels) rather than font weight.

Source: Existing codebase patterns -- `font-semibold text-xl` for h1, `text-sm` for body/table cells, `text-xs` for timestamps and error messages.

### Admin Lead Card (mobile < 768px)

```
+--------------------------------------------------+
| Nome do Lead                    [Tag Badge] [...] |
| Vendedor: Fulano                                  |
| (11) 99999-9999                                   |
+--------------------------------------------------+
```

- Card uses `p-4` (16px padding)
- Name: `text-sm font-semibold`, truncate at 1 line
- Tag badge: inline, right-aligned next to name
- Vendedor: `text-sm text-muted-foreground`
- Contact (phone or email): `text-sm text-muted-foreground`
- DropdownMenu trigger: `MoreVertical` icon (lucide `MoreVertical` or `Ellipsis`), positioned top-right, `min-h-[44px] min-w-[44px]`
- Fields hidden on mobile: Segmento, Data de criacao, Empresa
- Card gap: `gap-4` (16px) between cards in list

### Admin User Card (mobile < 768px)

```
+--------------------------------------------------+
| Nome do Usuario    [Role Badge]            [...] |
| 42 leads                     [Status Badge]      |
+--------------------------------------------------+
```

- Card uses `p-4` (16px padding)
- Name: `text-sm font-semibold`, truncate at 1 line
- Role badge: `Admin` (primary) or `Vendedor` (secondary)
- Lead count: `text-sm text-muted-foreground`
- Status badge: `Ativo` (green) or `Desativado` (destructive)
- DropdownMenu trigger: `MoreVertical` icon, top-right, `min-h-[44px] min-w-[44px]`
- Field hidden on mobile: Email
- Card gap: `gap-4` (16px) between cards in list

### Lead Form Grid (responsive)

```
Mobile (< 768px):                Desktop (>= 768px):
+---------------------------+    +-------------+-------------+
| Nome *              [full] |    | Nome *      | Telefone    |
+---------------------------+    +-------------+-------------+
| Telefone       [QR] [full] |    | Email       | Interesse * |
+---------------------------+    +-------------+-------------+
| Email                [full] |    | Foto        |             |
+---------------------------+    +-------------+-------------+
| Interesse *          [full] |    | [Mais detalhes]           |
+---------------------------+    +------ 2 cols ------+------+
| Foto                 [full] |    | Empresa     | Cargo      |
+---------------------------+    +-------------+-------------+
| [Mais detalhes]            |    | Segmento    | Notas      |
|   Empresa            [full] |    +-------------+-------------+
|   Cargo              [full] |    | [Salvar Lead]       [full] |
|   Segmento           [full] |    +---------------------------+
|   Notas              [full] |
+---------------------------+
| [Salvar Lead]        [full] |
+---------------------------+
```

- Form container: remove `max-w-[480px]` constraint on mobile, keep it on `md:` and above
- Grid: `grid grid-cols-1 md:grid-cols-2 gap-4`
- Required fields (Nome, Telefone+QR, Email, Interesse, Foto): `col-span-1` on both breakpoints (stack naturally)
- Collapsible content fields: `col-span-1` per field (2 columns on desktop)
- Submit button: `md:col-span-2` to span full width on desktop
- Textarea (Notas): `md:col-span-2` to span full width on desktop

### FAB Behavior

- Position: `fixed right-6 bottom-6 z-50` (existing)
- Size: `h-14 w-14` (56px, existing)
- Visibility: Only on `/leads` and `/dashboard` routes
- Keyboard hide: Listen to `window.visualViewport?.resize` event. When `visualViewport.height < window.innerHeight * 0.75`, set `display: none` on FAB. Restore when viewport height recovers
- No overlap with sidebar trigger (sidebar trigger is top-left, FAB is bottom-right)

### IntersectionObserver Fix

- Current: `new IntersectionObserver(callback, { rootMargin: "200px" })` -- uses viewport as root
- If SidebarInset is the scroll container (overflow-y: auto), set `root` to the SidebarInset element
- Detection: Check if `document.documentElement.scrollHeight === document.documentElement.clientHeight` (no document scroll). If true, find scrollable ancestor of sentinel and set as `root`

## Responsive Breakpoints

| Breakpoint | Tailwind | Layout |
|------------|----------|--------|
| < 768px (mobile) | default | Cards instead of tables, single-column form, FAB visible, sidebar as drawer |
| >= 768px (tablet+) | `md:` | Tables visible, 2-column form grid, sidebar visible inline |

Strategy: Mobile-first. Default styles = mobile. `md:` prefix = tablet and desktop. Matches existing `useIsMobile()` hook breakpoint at 768px.

Table/Card switch: Use `hidden md:block` on Table wrapper and `md:hidden` on Card list wrapper. Both components render in DOM; CSS controls visibility. This avoids layout flash from `useIsMobile()` hydration mismatch.

## Interaction Contracts

### Touch Targets

All interactive elements must have minimum 44x44px touch target:
- DropdownMenu trigger on cards: `min-h-[44px] min-w-[44px]` with `items-center justify-center`
- DropdownMenu items: shadcn default height is sufficient (40px+padding); verify >= 44px
- Pagination buttons: existing sizing sufficient on desktop; verify on mobile
- FAB: 56x56px (exceeds requirement)
- Form inputs: shadcn default height (40px) with 44px tap area via padding

### Destructive Actions

All destructive actions use AlertDialog confirmation (existing pattern):
1. **Excluir Lead** -- `AlertDialogAction` with `bg-destructive text-white` styling
2. **Desativar Usuario** -- `AlertDialogAction` with `variant="destructive"`
3. No destructive actions are accessible via single tap -- always requires confirmation dialog

### Card vs Table Switch

- No animation between card and table layouts
- Both layouts render in DOM simultaneously; CSS `hidden`/`md:block` controls visibility
- Avoids hydration mismatch from `useIsMobile()` returning `false` on SSR initial render

### FAB Keyboard Dismiss

- `visualViewport` API detects virtual keyboard presence
- FAB hides instantly (no animation) when keyboard opens
- FAB reappears instantly when keyboard closes
- Fallback for browsers without `visualViewport`: FAB stays visible (acceptable degradation)

## Accessibility

- DropdownMenu trigger: `aria-label="Abrir menu de acoes"` on each card's trigger button
- Card list: `role="list"` on container with cards as list items where applicable
- Pagination: existing `aria-disabled` pattern maintained
- Empty states: use `Empty`/`EmptyTitle`/`EmptyDescription` compound components (semantic headings)
- Error messages: `role="alert"` on form validation errors (existing pattern)
- All admin table headers preserved as `<th>` in desktop view
- Mobile cards are NOT interactive (no `role="button"` on admin cards) -- actions are via DropdownMenu only
