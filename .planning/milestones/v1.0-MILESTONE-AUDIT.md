## Cross-Phase Integration Findings

### Blockers

**AUTH-05 — Middleware naming bug**
- File: `apps/web/proxy.ts`
- Expected: `apps/web/middleware.ts` (Next.js convention)
- Impact: Cookie-based session refresh via `setAll` never fires. Token expiry not handled at middleware layer. Path-based unauthenticated redirect guard never runs.
- Fix: Rename `proxy.ts` → `middleware.ts`, rename export `proxy` → `middleware` (or re-export as default)

**ADMN-07 — Admin vendor dashboard always empty**
- Files: `apps/web/src/app/dashboard/dashboard.tsx`, `apps/web/src/lib/lead/stats.ts`
- Impact: Admin selects a vendor → `PersonalDashboard(userId=vendorId)` → `getPersonalStats(vendorId)` queries Dexie → admin has no other vendor's leads → always shows zeros
- Fix: When `isAdmin && selectedVendor !== currentUserId`, route stats through `admin.stats` tRPC query (server-side) instead of local Dexie `getPersonalStats()`

### Non-Blocking Issues

| Issue | Severity | Location | Description |
|-------|----------|----------|-------------|
| `/leads/new` no server auth redirect | Medium | `app/leads/new/page.tsx` | Client renders before auth check; other pages have server-side redirect |
| `isAdmin` from `app_metadata` vs `getClaims()` | Medium | `dashboard/page.tsx` vs `admin/layout.tsx` | Inconsistent role source; vendor selector may not appear for admins depending on Supabase config |
| Leaderboard shows "Vendedor" for all non-current users | Low | `packages/api/src/routers/leaderboard.ts` | JOIN to auth.users missing; admin.stats.getRanking has correct names but vendedor-facing leaderboard does not |
| ADMN-06/ADMN-07 missing from SUMMARY frontmatter | Low | `06-05-SUMMARY.md` | Frontmatter not updated; implementation exists in codebase |
