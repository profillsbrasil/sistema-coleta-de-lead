  <action>
    1. Create `apps/web/src/lib/sync/photo-upload.ts` (per D-13, CAPT-06, Research Pattern 4):
       - Function uploadPendingPhotos(): Promise<void>
       - Get Supabase client via createClient()
       - Query db.leads.filter: photo !== null AND serverId !== null
       - For each candidate:
         - Upload to supabase.storage.from("lead-photos").upload(`${lead.userId}/${lead.localId}.jpg`, lead.photo, { contentType: "image/jpeg", upsert: true })
         - On error: continue (retry next cycle)
         - On success: get public URL via getPublicUrl()
         - Add syncQueue update operation: { localId, operation: "update", payload: JSON.stringify({ photoUrl: publicUrl }), retryCount: 0, timestamp: now }
         - Clear local blob: db.leads.update(lead.localId, { photo: null })
       - Guard: if no candidates, return early

       - In syncCycle(), after pushChanges() and before pullChanges(), add:

