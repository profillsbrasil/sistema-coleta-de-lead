For "update": updates fields where localId matches AND userId matches. For "delete": sets deleted_at where localId matches AND userId matches.

  <behavior>
    - createConnectivityDetector(pollIntervalMs = 30_000) returns { start, stop, subscribe, isOnline }
    - start(): registers window online/offline event listeners + starts polling interval
    - stop(): clears interval, removes listeners, clears subscriber set
    - subscribe(fn): adds listener, returns unsubscribe function
    - isOnline: getter returning current connectivity state
    - Polling: HEAD request to /api/trpc/healthCheck with cache-busting query param. If response.ok -> online, catch -> offline
    - Notification: only fires listeners when state CHANGES (not on every poll)
    - SYNC_CONFIG: maxRetries=5, baseDelayMs=1000, maxDelayMs=30000, pollIntervalMs=30000
    - getBackoffDelay(retryCount): baseDelay * 2^retryCount + random jitter, capped at maxDelay
  </behavior>

       - Export `getBackoffDelay(retryCount: number): number` function: `Math.min(SYNC_CONFIG.baseDelayMs * 2 ** retryCount + Math.random() * SYNC_CONFIG.baseDelayMs, SYNC_CONFIG.maxDelayMs)` (per D-05: exponential backoff)
